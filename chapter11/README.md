# One Day One JavaScript 7

다음 요구사항에 따라 **랜덤 난수 생성기(Random Code Generator)**를 구현하시오.
화면에는 “난수 생성” 버튼이 있으며, 버튼을 클릭할 때마다 알파벳 대문자와 숫자가 조합된 형태의 랜덤 코드를 생성하고 화면에 출력한다.

## 상세 요구사항
- 이전 ‘랜덤 난수 생성기’ 과제와 동일
- 별도 파라미터 또는 옵션을 통해 규칙이 있는 코드 형식을 지정하지 않고, 클릭 시마다 랜덤하게 배치된 난수를 생성하도록 한다.
- 생성된 난수는 배열로 관리하며, 해당 배열의 내용을 DOM에 출력한다.
   - 중복된 값이 있을 경우 : alert등을 통해 중복 출력 및 재실행
   - 중복되지 않은 값일 경우 : 배열에 추가

## STUDY
### 함수 반환값으로 다른 함수 호출하기 : 함수 체인
- 동작 방식 : 중첩 함수 호출
  1. ```getRandomCode(count)``` 함수 실행
  2. 이 함수는 코드를 생성하는 작업을 완료 후, 결과를 반환해야 하는 시점에 ```this.getCodeDuplidate(code)``` 함수를 호출
  3. ```this.getCodeDuplidate(code)``` 함수가 실행되고, 중복 체크 및 배열 저장 작업 수행
  4. ```this.getCodeDuplidate``` 함수가 최종 결과 값(codeArr 또는 null)을 반환
  5. ```getRandomCode``` 함수는 ```getCodeDuplicate```가 반환한 이 최종 결과 값을 다시 자신의 최종 결과 값으로 호출자에게 반환하며 종료
-> 내 역할 끝났으니, 다음 역할(함수)를 실행시키고 그 결과를 그대로 나를 호출한 곳으로 전달해!

- 코드 설계 방식 : 함수 체인
  ᄂ 구조 : ```getRandomCode``` -> ```getCodeDuplicate``` -> ```randomCreate``` 순서로 결과가 연속적으로 전달되는 구조
  ᄂ 목적 : 한 함수의 결과를 다음 함수의 입력으로 사용해, 여러 작업을 연속적으로 수행하도록 연결
  ex)
    - ```getRandomCode``` : code 생성
    - ```getCodeDuplicate``` : 중복 체크 및 저장
    - ```return``` : 두 함수가 연결되어, 생성된 code가 바로 다음 단계(체크/저장)로 넘어가고 그 결과가 다시 최종적으로 전달 됨

### 문자열 결합
```+=``` 연산자로 문자열 결합시, 성능상 비효율적
```slide()```와 ```join()``` 배열메서드를 사용해 문자열 결합

### ```return```
```return```은 명확한 값을 반환하도록 지정해주는게 좋다
```undefined```로 반환은 좋지 않음

### 유효성검사 위치
유효성검사는 함수안에서 보다 이벤트발생되는 위치에서 동작하는게 역할분리, 기능명확화에 좋음
