# One Day One JavaScript 16

다음 요구사항에 따라 **날짜 유효성 검사기 (Date Validator)**를 구현하시오.
"2025-10-21 18:00:00" 형태의 날짜 문자열을 입력받아, 입력된 값이 올바른 날짜 및 시간 형식인지 검사하는 알고리즘을 구현한다.
(예: 2025-02-29 25:10:00 → 잘못된 날짜/시간으로 판단)

상세 요구사항
- 모든 동작은 순수 자바스크립트로 구현한다.
- 날짜 입력은 ```<input>``` 또는 ```prompt()```를 통해 문자열로 입력받는다.
- 입력값이 yyyy-mm-dd hh:mm:ss 형식에 맞는지 정규식으로 1차 검증
- 실제 날짜 객체(```new Date()```)로 변환하여 존재 가능한 날짜인지 2차 검증
- 유효한 경우 "유효한 날짜입니다.", 잘못된 경우 "잘못된 형식 또는 존재하지 않는 날짜입니다." 메시지 출력
- 검색을 통해 스스로 해결하며, AI를 활용해 코드를 생성하거나 회신받는 행위는 금지한다.

# STUDY
- 정규식 형식 표현 방법 : /^\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}$/
- 정규식 형태 검증 방법 : regExp.test();
- 날짜가 아닌 경우, Invalid Date 출력됨
- new Data() 객체 사용시, 자동으로 계산되어 평년일때 없는 날짜임에도 다음날짜로 자동 계산되어짐

# 구현완료
new Data() 객체 사용시, 자동으로 계산되어 평년일때 없는 날짜임에도 다음날짜로 자동 계산되어지는 것을 알게되었습니다.
input에서 받은 날짜문자열을 년월일시분초로 분리해 new Date() 객체에 넣고 그 값으로 날짜가 존재하는지 검증하는 것으로 구현했습니다.
